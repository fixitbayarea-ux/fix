# FixitBay.net - Product Requirements Document

## Original Problem Statement
React SPA for appliance repair business (FixitBay). SSG via custom script. SEO optimization (Ahrefs/GSC), mobile UX.

## Architecture
- React SPA + custom SSG (generate-seo-snapshots.cjs + seo-config.cjs)
- react-router-dom with useDeferredValue
- Netlify hosting with _redirects for 301s
- sitemap.xml auto-generated by postbuild
- **CRITICAL**: `_redirects` is generated by SSG script (generate-seo-snapshots.cjs), NOT from `public/_redirects`. All redirect changes MUST go in the SSG script.

## Current Status (Apr 2026)

- `/wall-oven-repair` is standalone indexable page
- All 301 redirects are now server-side via SSG-generated `_redirects`

## P13 Details — ROOT CAUSE FIX
- **Bug found**: SSG script placed city alternate-slug redirects AFTER `/* /index.html 200` catch-all. Netlify never matched them → SPA loaded → client-side `<Navigate>` redirected. Google saw 200+JS redirect, not 301.
- **Fix**: Moved `cityRedirects` and `manualRedirects` BEFORE the SPA catch-all. Added missing redirects (buggy city×city, maintenance paths, Marin pages, legacy aliases).
- **Canonical URLs**: Verified `canonical=` is set correctly in CityRepairPage.js template.
- **SSG snapshots**: All 7 in-service-area city pages present in seo-config.cjs routes.
